## Things I Learned

- Linux is completely case sensitive — `CD` and `cd` are different. I made sure to always use lowercase for commands.
- `ls -la` reveals hidden files that start with a dot like `.bashrc` and `.bash_logout` which are normally invisible with just `ls`.
- The `tree` command is a great way to visualize the entire folder structure in one view — much better than navigating folder by folder.
- `rm` has no recycle bin — once a file is deleted it is permanently gone. I need to be very careful with this command.
- `mv` serves two purposes — it can both move a file to a different folder and rename a file. I found this very useful.
- `touch` creates empty files instantly and can create multiple files at once in a single command.
- `history` is helpful when you forget what you typed earlier and want to re-run a command.
- The terminal prompt shows your username, hostname and current directory all at once which is very informative.

## Observations

- Kali Linux boots up quickly and the XFCE desktop environment feels smooth even in a virtual machine.
- The terminal is far more powerful than Windows Command Prompt — you can do complex tasks with a single line.
- Creating a structured folder like CyberSecurity_Lab made me realise how important organisation is in Linux.
- File permissions shown in `ls -la` like `drwxr-xr-x` are more meaningful now that I understand what each character represents.
